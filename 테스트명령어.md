# 테스트 명령어 정리

이 문서는 `/workspaces/B_PLUS-TREE` 기준으로 자주 쓰는 테스트 명령을 하나로 합쳐 정리한 문서다.

기준 환경은 아래와 같다.

- 작업 경로: `/workspaces/B_PLUS-TREE`
- 기본 테이블: `users`
- 기본 데이터 규모: `1000000`건
- 벤치 실행 파일: `./test_perf`, `./test_perf_sim`
- SQL 실행 파일: `./sqlp`, `./sqlp_sim`

현재 벤치마크에서 고정된 범위는 아래와 같다.

- 기본 `id-range`: `WHERE id BETWEEN 2500 AND 5000`
- 큰 `id-range`: `WHERE id BETWEEN 100000 AND 500000`
- `age-range`: `WHERE age BETWEEN 30 AND 40`

## 2. 100만 건 데이터 준비

벤치마크를 100만 건 기준으로 돌리려면 먼저 데이터를 생성하고 적재한다.

```bash

./sqlp samples/bench_users.sql >/tmp/load_out.txt 2>/tmp/load_err.txt

tail -n 20 /tmp/load_err.txt
```

큰 `id-range` 테스트는 `100000 ~ 500000`을 조회하므로, 데이터는 최소 50만 건 이상 있어야 의미가 있다. 이 문서의 기본 예시는 전부 100만 건 기준으로 맞춘다.

## 3. test_perf 실행 방식

`test_perf`와 `test_perf_sim`은 세 번째 인자로 실행할 항목을 고를 수 있다.

```bash
./test_perf [table] [rows] [all|point|id-range|id-range-large|age-range|height]
./test_perf_sim [table] [rows] [all|point|id-range|id-range-large|age-range|height]
```

각 항목 의미는 아래와 같다.

- `all`: 전체 테스트 실행
- `point`: 단건 조회 `WHERE id = rows/2`
- `id-range`: 고정 범위 조회 `WHERE id BETWEEN 2500 AND 5000`
- `id-range-large`: 고정 대범위 조회 `WHERE id BETWEEN 100000 AND 500000`
- `age-range`: `WHERE age BETWEEN 30 AND 40`
- `height`: `order=4`와 `order=128` 높이 비교

전체 실행 시 출력 순서는 아래와 같다.

- 테스트 1: 단건 조회
- 테스트 2: 범위 조회(id)
- 테스트 3: 대범위 조회(id)
- 테스트 4: 범위 조회(age)
- 테스트 5: 높이 비교

## 12. 최종 체크리스트

- `./sqlp`에서 `id = ...`가 `index:id:eq`로 가는지
- `--force-linear`에서 `id BETWEEN`, `age BETWEEN`이 정상 동작하는지
- `--compare`에서 기본 실행 결과와 선형 스캔 결과가 둘 다 출력되는지
- `sqlp_sim`에서 인덱스 경로는 `tree_io > 0`, linear는 `tree_io = 0`인지
- `./test_perf users 1000000 all`에서 5개 테스트가 모두 출력되는지
- 큰 `id-range`가 `100000 ~ 500000` 기준으로 나오는지
